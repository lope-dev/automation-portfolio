Agentic Jira Security Triage with LLM Summary

A fully automated security triage pipeline that eliminated manual write-ups across the entire Jira security queue. A cron trigger fires on schedule, pulls open Jira issues, splits them into batches, extracts the service name, looks up the responsible manager in Google Sheets, routes by priority, calculates the remediation date, reassigns the ticket, changes status to Triage, and then hands off to an LLM agent that writes a professional summary comment and posts it directly back to the Jira issue. Zero human steps required after deployment.

Cron Trigger-> Fetch Jira Issues-> Batch Split-> Manager Lookup-> Priority Check-> Assign + Remediation Date-> LLM Agent Summary-> Post Jira Comment
Cron-scheduled ingestion with batch processing so large Jira queues never time out
Google Sheets used as a dynamic manager lookup table, no hardcoded assignments
LLM agent writes professional triage summaries from raw ticket data and posts them as Jira comments automatically
Remediation date calculation logic runs per-issue based on priority level before status change

Contractor Missed Call Lead Recovery System

Most contractors lose leads the moment a call goes unanswered. This system catches every one. A missed call trigger fires instantly, logs the lead in Airtable, sends an immediate SMS text-back within seconds, then runs a structured follow-up sequence: waits 10 minutes, checks if the lead replied, qualifies them with AI if they did, sends a booking link, and alerts the contractor by SMS. If no reply, it escalates through 2-hour and 24-hour follow-ups before marking the lead cold. After a job completes, it automatically sends a review request. A monthly re-engagement scheduler loops back through past cold leads.

Missed Call-> Log in Airtable-> Instant SMS-> Wait + Check Reply-> AI Qualify-> Book / Follow-Up / Cold
Three trigger types: missed call, job completion, and monthly re-engagement schedule, each running its own path
Airtable as the lead CRM with real-time status updates at every stage of the funnel
AI lead qualification decides whether to send a booking link or continue nurturing based on reply content
Full lifecycle from first missed call to job completion review request and long-term re-engagement

Multi-Agent Classroom Schedule Optimizer with GPT-4o and Google Calendar

The most architecturally complex workflow in this portfolio. A scheduled trigger kicks off a four-agent pipeline: a Master Orchestrator coordinates three specialized agents, each with their own GPT-4o model and structured output parser. The Resource Analysis Agent examines room and instructor availability. The Operations Agent identifies scheduling conflicts. The Optimization Agent proposes resolutions. Each agent produces structured JSON output, results are merged, conflict severity is scored, auto-resolution is attempted where possible, and then critical conflicts trigger Slack alerts, while recommendations are emailed as a full report. Google Calendar is used as a live tool throughout.

Schedule Trigger-> Master Orchestrator-> Resource Agent| Operations Agent| Optimization Agent-> Conflict Score-> Slack + Email Report
Four specialized GPT-4o agents each with their own model node and structured output parser for typed JSON responses
Master Orchestrator agent coordinates agent sequence, handles inter-agent context passing, and makes routing decisions
Google Calendar Tool node gives agents read/write access to live calendar data mid-reasoning
Conflict scoring, auto-resolution attempt, Slack critical alert, and full email report all in one automated run
34 nodes, the most complex workflow here, and a direct analogue for any resource scheduling problem in healthcare, logistics, or enterprise operations
